# Claude's working rules

## Workflow

1. **Write tests first** (TDD) for any new logic
2. **Run tests** — all must pass before presenting work for review
3. **Present for review** — show the user what was built and the test results
4. **Push only after explicit approval**

## Commits

- One logical change per commit
- Always ask before committing unless the user explicitly says to push

## Docs

- Keep `INTEGRATION.md` in sync with any data model or API changes
